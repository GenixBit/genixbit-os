1. Overview

GenixBit OS is an AI-first, local-first Linux operating system engineered for developers, creators, server managers, and AI application builders. It provides a secure, predictable, and hardware-accelerated workstation base built upon Ubuntu 26.04 Resolute development base.

Key Philosophy: Local-first AI workflows, strict package signing, zero unverified cloud telemetry, and transparent open-source engineering.

3. Installation & Bootable USB Creation

GenixBit OS requires a 64-bit amd64 architecture. Secure Boot setup should be disabled or set to Audit mode for custom kernel drivers.

Creating a Bootable USB Flash Drive

Warning: The dd command will overwrite all data on the target USB device. Confirm device path carefully.
# Identify your USB drive device path (e.g., /dev/sdX) sudo lsblk # Write ISO to USB drive sudo dd if=GenixBitOS-0.2.0-alpha-2607220558.iso of=/dev/sdX bs=4M status=progress conv=fsync

4. Virtual Machine Testing Guide

To test GenixBit OS inside a QEMU/KVM virtual machine:

qemu-system-x86_64 \ -enable-kvm \ -m 4096 \ -smp 4 \ -cpu host \ -cdrom GenixBitOS-0.2.0-alpha-2607220558.iso \ -boot d

6. Branding Architecture

GenixBit OS enforces consistent visual identity across GRUB bootloader, Plymouth splash, terminal banners, desktop artwork, and /etc/os-release while preserving legally mandated upstream copyright notices.

7. Package Architecture

The package ecosystem features dual-suite channels (resolute-alpha and resolute-testing), automated dpkg-scanpackages indexes, and atomic directory symlinking for instant publication.

8. GCP Staging Validation

We executed a full 12-stage package repository validation drill on an isolated GCP staging infrastructure under run ID run-staging-20260723-001.

Staging Notice: The GCP staging exercise was purely for validation. All temporary resources (VPC, VMs, NAT, DNS, buckets) were destroyed. Production repo remains undeployed.

9. Signing & Trust Model

OpenPGP metadata signatures (InRelease and Release.gpg) are produced exclusively on an isolated Signing Workstation (SIGNER_INSTANCE_NAME). Repository Host and APT Client nodes contain zero secret key packets.

10. Snapshots & Rollback

Supports atomic release snapshots (e.g., snap-resolute-alpha-20260723-104011) allowing instant rollbacks to previous validated suite states without client-side APT corruption.

11. Security Policy

Security vulnerabilities should be reported directly to security@genixbit.com. We enforce zero hardcoded secrets, fail-closed operational runners, and anti-fabrication testing gates across all CI pipelines.

12. Troubleshooting

If you encounter boot issues, verify ISO SHA-256 hash, disable Secure Boot in BIOS, or check QEMU memory allocation (minimum 4 GB RAM required).
