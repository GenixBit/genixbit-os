1. Overview

GenixBit OS is an AI-first, local-first Linux operating system engineered for developers, creators, server managers, and AI application builders. It provides a secure, predictable, and hardware-accelerated workstation base built upon Ubuntu 26.04 Resolute development base.

Key Philosophy: Local-first AI workflows, strict package signing, zero unverified cloud telemetry, and transparent open-source engineering.

2. Installation

GenixBit OS is distributed as a bootable hybrid ISO image suitable for bare-metal x86_64 hardware, KVM/QEMU virtual machines, and modern hypervisors.

System Requirements

  • CPU: 64-bit x86_64 processor (quad-core recommended)
  • RAM: 8 GB minimum (16 GB+ recommended for local LLM inference)
  • Storage: 32 GB NVMe/SSD space
  • GPU: Optional NVIDIA or AMD GPU for hardware-accelerated model execution

5. Branding Architecture

GenixBit OS enforces consistent visual identity and system metadata across terminal banners, desktop artwork, GRUB bootloader, Plymouth splash screens, and /etc/os-release while preserving legally mandated upstream copyright attributions.

6. Package Architecture

The GenixBit package ecosystem features dual-suite release channels (resolute-alpha and resolute-testing), automated dpkg-scanpackages index generation, and atomic directory symlink swappings for instant production publication.

7. GCP Staging Validation

We executed a full 12-stage package repository validation drill on an isolated GCP staging infrastructure under run ID run-staging-20260723-001.

Staging Transparency Notice: The GCP deployment exercise was purely for staging validation. All temporary resources (VPC, VMs, Cloud NAT, DNS, and Storage Buckets) were completely torn down and cleaned up upon completion. The production package repository remains undeployed.

8. Signing & Trust Model

OpenPGP metadata signatures (InRelease and Release.gpg) are produced exclusively on a isolated Signing Workstation (SIGNER_INSTANCE_NAME). Repository Host and APT Client nodes contain zero secret key packets.

9. Snapshots & Rollback

Staging supports atomic release snapshots (e.g., snap-resolute-alpha-20260723-104011) allowing instant rollbacks to previous validated suite states without client-side APT corruption.

10. Security Policy

Security vulnerabilities should be reported directly to security@genixbit.com. We enforce zero hardcoded secrets, fail-closed operational runners, and complete anti-fabrication testing gates across all CI pipelines.

11. Troubleshooting

For common build, GPG key import, or repository sync issues, consult our automated test suite logs or run preflight diagnostics using ./preflight.sh in the repository root.
