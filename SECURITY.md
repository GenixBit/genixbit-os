# Security Policy

## Early Development Notice

> [!CAUTION]
> **GenixBit OS** is currently in **early alpha status (`0.1.0-alpha`)**.
> It is intended exclusively for development, testing, and evaluation. **Do NOT use early alpha builds on production machines, critical infrastructure, or systems handling sensitive data.**

---

## Reporting a Security Vulnerability

We take the security of GenixBit OS seriously. If you discover a security vulnerability or security defect in GenixBit OS, please report it responsibly.

### Responsible Disclosure Guidelines

* **Do NOT report security vulnerabilities containing exploit details or sensitive security findings via public GitHub issues, public pull requests, or public forums.**
* To report a vulnerability privately, use **[GitHub Private Vulnerability Reporting](https://github.com/GenixBit/genixbit-os/security/advisories/new)** via the repository's Security Advisories tab.
* If private reporting is unavailable, contact the project security maintainers directly via official security communication channels before disclosing details publicly.

---

## Response Process

* Upon receiving a private security report, maintainers will review the submission and investigate the reported issue.
* Validated security vulnerabilities will be addressed in a subsequent release or patch update.
* Credit will be given to security researchers who report vulnerabilities responsibly following these guidelines.
